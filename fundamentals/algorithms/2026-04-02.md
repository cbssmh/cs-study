# Sorting Overview

Sorting algorithms are compared based on:

- Time Complexity
- In-place 여부
- Stability (안정 정렬 여부)

---

# Basic Sorting Algorithms

## Selection Sort

- Find minimum and place at front
- Time Complexity: O(n²)
- Not stable

---

## Bubble Sort

- Compare adjacent elements and swap
- Stable sort
- Time Complexity: O(n²)

---

## Insertion Sort

- Assume left side is sorted
- Insert current element into correct position

### 특징

- Best: O(n) (이미 정렬된 경우)
- Worst: O(n²)
- Stable sort
- Nearly sorted data에서 매우 빠름

---

# Heap Sort

A sorting algorithm using a heap data structure.

---

## Heap Properties

1. Complete Binary Tree  
2. Heap Property  
   - Max Heap: parent ≥ children  
   - Min Heap: parent ≤ children  

---

## Heap Sort Process

1. Build Heap from array  
2. Extract root (max or min)  
3. Move last element to root  
4. Heapify  
5. Repeat  

---

## Heapify

- Assume subtrees are already heaps
- Compare parent with children
- Swap and recursively adjust

---

## 특징

- Time Complexity: O(n log n)
- In-place
- Not stable

---

# Shell Sort

Improved version of insertion sort.

---

## Idea

- Sort elements with a gap
- Gradually reduce gap
- Final step becomes insertion sort

---

## 특징

- Average: O(n^1.5)
- Worst: O(n²)
- Not stable
- Improves insertion sort performance

---

# Comparison vs Non-comparison Sort

## Comparison Sort

- Selection / Bubble / Insertion / Heap / Quick / Merge / Shell
- Based on element comparison

---

## Non-comparison Sort

Requires constraints

---

## Radix Sort

- Sort by digit (LSD → MSD)
- Uses stable sorting internally

### 특징

- Time Complexity: O(nk)
- Stable
- Only works for integers (or limited format)

---

## Counting Sort

- Count occurrences of each value

### 특징

- Time Complexity: O(n + k)
- Very fast if range is small
- Requires additional memory
- Range 제한 필요

---

# Greedy + Sorting

## Meeting Room Problem

### Idea

- Sort by end time
- Always pick the earliest finishing meeting

---

### Algorithm

1. Sort by end time  
2. Iterate  
3. Select if not overlapping  

---

### Time Complexity

O(n log n)

---

# Two Pointer + Sorting

## Two Liquids Problem

### Goal

- Find two values whose sum is closest to 0

---

### Idea

- Sort array
- Use two pointers

---

### Algorithm

1. left = 0, right = n-1  
2. Compute sum  
3.  
   - sum > 0 → right--  
   - sum < 0 → left++  
4. Track minimum  

---

### 특징

- Time Complexity: O(n log n)
- Avoids O(n²) brute force

# Key Insights

# Summary

- O(n²) → small data only
- O(n log n) → standard for large data
- Linear sort → only with constraints
