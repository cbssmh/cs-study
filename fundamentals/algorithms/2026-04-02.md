# Sorting Overview

Sorting algorithms are evaluated based on:

- Time complexity
- Whether they are in-place
- Stability (whether equal elements maintain their relative order)

---

# Basic Sorting Algorithms

## Selection Sort

- Repeatedly finds the minimum element and places it at the front
- Time Complexity: O(n²)
- Not stable

---

## Bubble Sort

- Repeatedly compares adjacent elements and swaps them if needed
- Time Complexity: O(n²)
- Stable

---

## Insertion Sort

- Assumes the left portion is already sorted
- Inserts each element into its correct position

### Characteristics

- Best Case: O(n) (already sorted)
- Worst Case: O(n²)
- Stable
- Very efficient for nearly sorted data

---

# Heap Sort

A sorting algorithm based on the heap data structure.

---

## Heap Properties

1. Complete binary tree  
2. Heap property  
   - Max heap: parent ≥ children  
   - Min heap: parent ≤ children  

---

## Heap Sort Process

1. Build a heap from the array  
2. Extract the root (max or min)  
3. Move the last element to the root  
4. Apply heapify  
5. Repeat  

---

## Heapify

- Assumes subtrees are already heaps
- Compares parent with children
- Swaps and recursively adjusts

---

## Characteristics

- Time Complexity: O(n log n)
- In-place
- Not stable

---

# Shell Sort

An improved version of insertion sort.

---

## Idea

- Sort elements with a certain gap
- Gradually reduce the gap
- Final step becomes standard insertion sort

---

## Characteristics

- Average: O(n^1.5)
- Worst: O(n²)
- Not stable
- Improves performance over basic insertion sort

---

# Comparison vs. Non-comparison Sorting

## Comparison-based Sorting

- Selection, Bubble, Insertion, Heap, Quick, Merge, Shell
- Based on comparing elements

---

## Non-comparison Sorting

Requires specific constraints on data.

---

## Radix Sort

- Sorts numbers digit by digit (LSD → MSD)
- Uses a stable sorting algorithm internally

### Characteristics

- Time Complexity: O(nk)
- Stable
- Works only for integers or constrained formats

---

## Counting Sort

- Counts occurrences of each value

### Characteristics

- Time Complexity: O(n + k)
- Very fast when the value range is small
- Requires additional memory
- Limited by value range

---

# Python Sorting

## sort()

- Sorts the original list in-place

## sorted()

- Returns a new sorted list

---

## key Option

```python
sorted(data, key=lambda x: (x[1], x[0]))
```

- Enables multi-criteria sorting

# Greedy + Sorting

## Meeting Room Problem

### Idea

- Sort meetings by end time
- Always select the meeting that finishes earliest

---

### Algorithm

1. Sort by end time  
2. Iterate through meetings  
3. Select if it does not overlap  

---

### Time Complexity

O(n log n)

---

# Two Pointers + Sorting

## Two Liquids Problem

### Goal

- Find two values whose sum is closest to 0

---

### Idea

- Sort the array
- Use two pointers

---

### Algorithm

1. Initialize left = 0, right = n - 1  
2. Compute sum  
3.  
   - If sum > 0 → decrease right  
   - If sum < 0 → increase left  
4. Track the minimum absolute sum  

---

### Characteristics

- Time Complexity: O(n log n)
- Avoids O(n²) brute-force approach
