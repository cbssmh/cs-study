# Graph Basics

A graph is a structure composed of:

- Vertices (nodes)
- Edges (connections)

It represents relationships between entities.

---

# Adjacency & Degree

## Adjacency

Two vertices are adjacent if they are connected by an edge.

Example:
- A — B → A and B are adjacent

## Degree

The number of edges connected to a vertex.

- Degree(A) = number of connections A has

---

# Connected vs Disconnected Graph

## Connected Graph

- All vertices are reachable from each other

## Disconnected Graph

- Some vertices are not connected

## Key Insight

- A graph does NOT have to be fully connected

---

# Cycle

A cycle is a path that starts and ends at the same vertex.

Example:
A → B → C → A

## Key Property

- Cycles must NOT exist in a tree

---

# Tree

A tree is:

- A connected graph
- With NO cycles

## Properties

- If there are N vertices → edges = N - 1
- Exactly one path between any two vertices

## Core Idea

Tree = minimal structure that keeps connectivity

---

# Spanning Tree

A spanning tree is:

- A tree that includes ALL vertices of a graph

## How to Create

- Remove edges from a graph
- Keep it connected
- Remove cycles

## Core Idea

Maintain connectivity while eliminating redundancy

---

# Minimum Spanning Tree (MST)

An MST is:

- A spanning tree
- With the minimum total edge weight

## Goal

- Connect all vertices
- Minimize total cost

---

# Kruskal Algorithm

Build MST by selecting edges.

## Process

1. Sort edges by weight
2. Pick smallest edge
3. If it creates a cycle → skip
4. Repeat until N-1 edges selected

## Key Tool

- Union-Find (for cycle detection)

---

# Prim Algorithm

Build MST by expanding from a vertex.

## Process

1. Start from any vertex
2. Select the smallest edge connected to current tree
3. Add new vertex
4. Repeat

## Core Idea

- Grow one connected component

---

# Kruskal vs Prim

## Kruskal

- Edge-based
- Works well for sparse graphs

## Prim

- Vertex-based
- Works well for dense graphs

---

# Union-Find

Used for grouping and cycle detection.

## Operations

### Find

- Identify the representative (root) of a set

### Union

- Merge two sets

---

## Why It Matters

- If two vertices belong to the same set
→ connecting them creates a cycle

---

# Key Exam Points

- Tree = connected + no cycle
- Edge count in tree = N - 1
- Spanning Tree = includes all vertices
- MST = minimum cost spanning tree
- Kruskal = edge selection + Union-Find
- Prim = expansion from a vertex
- Union-Find = cycle detection

# One-Line Summary

Graph = connection structure

Spanning Tree = connects all vertices  
MST = minimum cost connection

Prim = vertex-based greedy
