# Machine Learning Fundamentals

## Definition

Machine Learning is a subfield of Artificial Intelligence where a system improves its performance through experience (data).

> A program is said to learn from experience **E** with respect to tasks **T** and performance measure **P** if its performance at tasks in **T**, measured by **P**, improves with experience **E**.

---

# Artificial Intelligence vs Machine Learning vs Deep Learning

## Definition

### Artificial Intelligence (AI)
Broad concept of making machines perform intelligent behavior.

### Machine Learning (ML)
AI systems that improve through learning from data.

### Deep Learning (DL)
A subset of ML using multi-layer neural networks.

---

## Core Idea

```text
Artificial Intelligence
 └── Machine Learning
      └── Deep Learning
```

- AI does NOT necessarily require learning
- ML requires learning from experience/data
- DL specifically uses neural networks

---

## Notes

### AI without learning
Rule-based systems are AI but NOT ML.

Examples:
- Rule-based chatbot
- Spam filter using keyword rules

Example:

```python
if "FREE" in title:
    spam = True
```

→ Intelligent behavior  
→ No learning involved

---

# Supervised Learning

## Definition

Learning using labeled data.

Dataset structure:

```text
(X, Y)
```

- X → input
- Y → correct answer (label)

---

## Core Idea

Learn a function:

```math
f(X)=Y
```

Goal:
- predict output Y from input X

---

## Process

1. Prepare labeled dataset
2. Define model structure
3. Predict outputs
4. Compute loss
5. Update parameters
6. Repeat until performance improves

---

## Implementation

Typical tasks:

### Classification
Predict discrete categories.

Examples:
- cat vs dog
- spam vs normal
- disease classification

### Regression
Predict continuous numerical values.

Examples:
- house price prediction
- stock prediction
- weight estimation

---

## Key Insight

> Supervised Learning approximates an unknown "true mapping function" using parameterized models.

The real-world mapping is unknown.

ML builds an approximate function:

```text
f(x; θ)
```

where:
- θ = learnable parameters

---

## Complexity

Depends on:
- model size
- dataset size
- optimization method

---

## Notes

### Very Important

Supervised learning = learning from explicit answers.

---

# Unsupervised Learning

## Definition

Learning patterns from unlabeled data.

Dataset:

```text
X only
```

No labels exist.

---

## Core Idea

The system discovers:
- structure
- distribution
- similarity
- hidden patterns

---

## Process

1. Receive unlabeled data
2. Measure similarity
3. Group or compress data
4. Discover latent structure

---

## Implementation

Typical tasks:
- Clustering
- Anomaly Detection
- Dimensionality Reduction

---

## Key Insight

> The model learns relationships BETWEEN data points rather than learning explicit answers.

---

# Clustering

## Definition

Grouping similar data points together.

---

## Core Idea

Similar samples should belong to the same group.

Example:
- cat images cluster together
- dog images cluster together

without knowing actual labels.

---

# K-means Algorithm

## Definition

A clustering algorithm that partitions data into K clusters.

---

## Core Idea

Repeatedly:
- assign points to nearest center
- update centers using averages

until convergence.

---

## Process

### Step 1 — Initialize Centers
Randomly choose K centroids.

### Step 2 — Assign Points
Assign each point to nearest centroid.

### Step 3 — Update Centers
Compute average of assigned points.

### Step 4 — Repeat
Continue until centroids stop moving.

---

## Implementation

### Distance-based assignment

Usually Euclidean distance is used.

### Important
K must be manually specified.

---

## Key Insight

> K-means minimizes intra-cluster distance by iterative centroid updates.

---

## Complexity

For:
- N = number of points
- K = number of clusters
- I = iterations

Time Complexity:

```text
O(NKI)
```

---

## Notes

### Very Important

K-means may converge to:
- local optimum
- poor clustering

depending on initial centroid positions.

### Common Mistake

Different initializations can produce different results.

# Anomaly Detection

## Definition

Detecting abnormal or outlier data points.

---

## Core Idea

Normal data forms dense regions.

Outliers appear far from major distributions.

---

## Implementation

The model learns:
- what "normal" looks like
- whether new data deviates significantly

Applications:
- fraud detection
- network intrusion
- manufacturing defects

---

## Key Insight

> Outliers are identified through deviation from learned distributions.

---

---

## Core Idea

Preserve important structure while compressing representation.

---

## Implementation

Algorithms:
- PCA

Applications:
- visualization
- feature compression
- representation learning

# Reinforcement Learning

## Definition

Learning through interaction with an environment using rewards.

---

## Core Idea

An agent learns actions that maximize long-term reward.

---

## Process

```text
State → Action → Reward
```

1. Observe current state
2. Choose action
3. Receive reward
4. Update policy
5. Repeat

---

## Implementation

### State
Current environment snapshot.

Examples:
- robot joint angles
- game screen
- sensor values

### Action
What the agent does.

Examples:
- move motor
- jump
- turn left

### Reward
Quality measure of behavior.

Examples:
- distance traveled
- game score
- survival time

---

## Key Insight

> Reinforcement Learning optimizes behavior through reward maximization.

---

## Notes

### Very Important

RL does NOT learn from labeled answers.

It learns from:
- trial
- error
- reward signals

---

# Optimization

## Definition

Finding parameters that minimize or maximize an objective function.

---

## Core Idea

Machine Learning is fundamentally an optimization problem.

Goal:

```text
Find best θ
```

to optimize performance.

---

## Implementation

Usually:
- minimize loss
- maximize reward

---

## Key Insight

> Learning = parameter optimization.

---

# Loss Function

## Definition

A numerical measure of prediction error.

---

## Core Idea

Lower loss = better model performance.

Examples:
- classification error
- absolute difference
- squared error

---

## Notes

### Very Important

Loss functions define the learning objective.

Without loss:
- no optimization
- no learning

---

# Gradient Descent

## Definition

An optimization algorithm that minimizes loss using gradients.

---

## Core Idea

Move parameters in the direction where loss decreases fastest.

---

## Process

1. Compute gradient
2. Move opposite to gradient
3. Repeat iteratively

---

## Formula

```math
\theta_{t+1}=\theta_t-\eta\nabla L(\theta_t)
```

where:
- θ = parameters
- η = learning rate
- ∇L = gradient

---

## Implementation

### Gradient
Slope of the loss function.

### Learning Rate
Controls movement size.

- too large → divergence
- too small → slow convergence

---

## Key Insight

> Gradient indicates the steepest increase direction.  
> Gradient Descent moves in the opposite direction.

---

## Complexity

Depends on:
- number of parameters
- dataset size
- optimization iterations

---

## Notes

### Very Important

Gradient Descent is the foundation of neural network training.

### Common Mistake

Large learning rates may cause oscillation or divergence.

---
