# Greedy Algorithms & Huffman Coding

> **Very Important**
> Greedy algorithms make the **best local choice at the current moment**.
>
> Once selected, the choice is usually **not reverted**.

---

# Greedy Algorithm

## Definition

A strategy that solves problems by repeatedly selecting the locally optimal choice.

---

## Core Idea

- Focus on the **current best option**
- Do not consider all future cases
- Assumes:
  
  Local optimal choice  
  → leads to  
  Global optimal solution

---

## Greedy Characteristics

### Common Properties

- Select current best choice
- Never backtrack
- Fast implementation
- Often based on:
  - sorting
  - priority queue
  - local optimization

---

## Existing Greedy Examples

### Prim's Algorithm

## Core Idea

Select the minimum crossing edge between:

- Connected vertices group (A)
- Unconnected vertices group (B)

### Why Greedy?

- Only considers the smallest edge **at current state**
- Does not reconsider previous choices

---

### Kruskal's Algorithm

## Core Idea

- Sort edges by weight
- Select smallest edge
- Ignore edges creating cycles

### Why Greedy?

- Selects locally minimum edge
- Never changes previous selections

---

### Dijkstra Algorithm

## Core Idea

- Select vertex with smallest temporary distance
- Finalize shortest distance immediately

### Why Greedy?

- Chooses minimum current distance
- Never revisits finalized node

---

# Huffman Coding

## Definition

A greedy algorithm used for:

- text compression
- variable-length binary encoding

---

## Core Idea

Frequently appearing characters  
→ shorter bit codes

Rare characters  
→ longer bit codes

This minimizes total encoded size.

---

# Prefix Problem

## Problem

Suppose:

- A → `0`
- B → `1`
- C → `10`

Then:

```text
10
```

can mean:

- `C`
- `BA`

→ Decoding becomes ambiguous.

---

## Prefix

A code is a **prefix** of another code if:

```text
1 is prefix of 10
```

This causes decoding ambiguity.

---

> **Very Important**
>
> Huffman Coding creates
>
> **Prefix-Free Codes**
>
> so decoding becomes unique.

---

# Prefix-Free Example

Valid encoding:

- A → `0`
- B → `10`
- C → `11`

Now no code is prefix of another.

---

## Why Decoding Works

Example:

```text
0010011
```

Decoding:

- `0` → A
- `0` → A
- `10` → B
- `0` → A
- `11` → C

Unique decoding becomes possible.

---

# Huffman Tree

## Definition

Binary tree used to generate prefix-free binary codes.

---

## Core Idea

Repeatedly combine:

- two lowest-frequency nodes

using a priority queue.

---

## Process

### Step 1: Count Frequencies

Example:

| Character | Frequency |
|---|---|
| A | 3 |
| B | 2 |
| C | 1 |

---

### Step 2: Insert into Min Priority Queue

```text
C(1), B(2), A(3)
```

---

### Step 3: Repeatedly Merge

Take two smallest nodes:

```text
C(1), B(2)
```

Create:

```text
BC(3)
```

Tree:

```text
    BC(3)
   /    \
 C(1)  B(2)
```

Insert back into queue.

---

### Step 4: Repeat

Merge:

```text
A(3), BC(3)
```

Result:

```text
        ABC(6)
       /      \
     A(3)    BC(3)
             /   \
           C(1) B(2)
```

---

## Encoding Rule

- Left edge → `0`
- Right edge → `1`

---

## Final Codes

| Character | Code |
|---|---|
| A | 0 |
| C | 10 |
| B | 11 |

---

# Huffman Decoding

## Process

Start from root.

- `0` → go left
- `1` → go right

When leaf node reached:
- output character
- return to root

---

## Example

Encoded string:

```text
0011011
```

Traversal:

```text
0  -> A
0  -> A
11 -> B
0  -> A
11 -> B
```

---

## Key Insight

> Huffman coding guarantees:
>
> - efficient compression
> - unique decoding
> - prefix-free property

---

## Complexity

Using priority queue:

- Build Tree:
  
```text
O(n log n)
```

where `n` = number of unique characters.

---

# ATM Waiting Time Problem

## Definition

Minimize total waiting time of people using ATM.

---

## Core Idea

Smaller processing times should appear earlier.

Because:

Earlier values are added many times.

---

## Example

Times:

```text
3 1 4 3 2
```

Sort:

```text
1 2 3 3 4
```

---

## Why Sorting Works

First value contributes:

```text
N times
```

Second:

```text
N-1 times
```

...

Last:

```text
1 time
```

Therefore:

Smallest values must receive largest multipliers.

---

## Process

1. Sort ascending
2. Compute prefix sums
3. Sum all waiting times

---

## Complexity

Sorting:

```text
O(N log N)
```

---

# Number Binding Problem

## Definition

Group numbers to maximize final sum.

Grouped numbers are multiplied.

Ungrouped numbers are added normally.

---

## Core Idea

Different number types behave differently.

---

## Positive Numbers (>1)

### Rule

Largest positive numbers should be multiplied together.

Example:

```text
4 × 3 > 4 + 3
```

---

## Number 1

### Important

```text
1 × x = x
1 + x > x
```

So:

- Never multiply 1

---

## Negative Numbers

### Rule

Multiply negatives together.

Example:

```text
(-10) × (-9) = 90
```

Huge gain.

---

## Zero

Useful for neutralizing leftover negative number.

Example:

```text
(-5) × 0 = 0
```

---

## Strategy

### Positive Side

- Sort descending
- Pair numbers > 1

### Negative Side

- Sort ascending
- Pair negatives

### Ones

- Add directly

### Leftover Negative

- Pair with zero if possible

---

## Complexity

Sorting dominates:

```text
O(N log N)
```

---

# Rope Problem

## Definition

Each rope supports weight `w`.

Using `k` ropes:

Each rope bears:

```text
W / k
```

Need maximum possible weight.

---

## Core Idea

If using multiple ropes:

The weakest rope determines capacity.

---

## Key Formula

If sorted descending:

```text
rope[i] × (i+1)
```

---

## Why?

When using first `(i+1)` ropes:

- smallest rope among them limits weight
- all ropes share load equally

---

## Process

1. Sort descending
2. For each index:
   
```text
capacity = rope[i] × (i+1)
```

3. Take maximum

---

## Example

Sorted:

```text
15 10 8 7
```

Capacities:

| Ropes Used | Capacity |
|---|---|
| 1 | 15 |
| 2 | 20 |
| 3 | 24 |
| 4 | 28 |

Answer:

```text
28
```

---

## Complexity

Sorting:

```text
O(N log N)
```

# P Class

## Definition

Examples:

- sorting
- shortest path
- MST

## Polynomial Time

Examples:

```text
O(N)
```

# NP Class

## Definition

Problems where:

A given solution can be verified in polynomial time.

---

## Important Difference

Finding solution:
- may be hard

Checking solution:
- easy

---

# Example: SAT Problem

## Definition

Determine whether logical variables can satisfy a logical formula.

---

## Why NP?

Given candidate solution:

```text
x=1, y=0, z=1
```

We can verify quickly.

But finding it may require:

```text
2^N
```

search.

---

# NP-Hard

## Definition

Problems at least as hard as NP problems.

Includes:
- NP-complete
- even harder problems

---

# NP-Complete

## Definition

Hardest problems inside NP.

If one NP-complete problem is solved in polynomial time:

```text
P = NP
```

---

# A* Algorithm

## Definition

Heuristic extension of Dijkstra.

---

## Core Idea

Dijkstra:

```text
f(x) = g(x)
```

A*:

```text
f(x) = g(x) + h(x)
```

Where:

- `g(x)` = actual distance
- `h(x)` = estimated remaining distance

---

## Key Insight

A* sacrifices perfect optimality slightly  
→ gains huge speed improvement.

---

# Heuristic Optimization Methods

Inspired by evolution.
