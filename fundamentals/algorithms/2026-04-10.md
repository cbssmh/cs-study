# Search Algorithms

Search algorithms explore a space to find a target or solution.

Types:

- Linear Search
- Complete Search (Brute Force)
- Graph Search (DFS, BFS)

---

# DFS (Depth-First Search)

DFS explores as deep as possible before backtracking.

## Characteristics

- Uses stack (or recursion)
- Path-dependent
- Exploration order affects performance

## Pros

- Simple implementation
- Low memory usage

## Cons

- Does not guarantee shortest path
- Can be inefficient depending on priority

---

# BFS (Breadth-First Search)

BFS explores nodes level by level.

## Characteristics

- Uses queue
- Explores same-distance nodes together

## Pros

- Guarantees shortest path (minimum steps)
- Stable regardless of traversal order

## Cons

- Higher memory usage

---

# DFS vs BFS

## DFS
- Order-sensitive
- Can be fast or very slow
- Not optimal for shortest path

## BFS
- Distance-based exploration
- Always finds shortest path

---

# Maze Search

Search order example:

R → D → L → U

## DFS Behavior
- Follows one path deeply
- Backtracks when blocked

## BFS Behavior
- Expands all directions evenly
- Finds shortest path

---

# Priority Impact

Changing traversal order (e.g., RDLU → ULDR):

- DFS → result changes drastically
- BFS → result mostly stable

---

# Shortest Path Strategy

Use BFS when:

- Minimum distance required
- Minimum time required
- Minimum number of moves required

---

# Micro Mouse

A robot solving a maze autonomously.

## Strategies

- DFS → explores deeply but inefficient
- BFS → finds shortest path but redundant

---

# Flood Fill Algorithm

Used in Micro Mouse competitions.

## Idea

- Maintain distance map
- Move toward decreasing distance values
- Update map when walls are discovered

## Features

- Not guaranteed optimal in first run
- Improves with repeated exploration

---

# Practical Insight

Shortest path ≠ fastest path

- Fewer turns can be faster
- Physical constraints matter

---

# State Space Search

Used in problems like:

- Hide and Seek (N → K)
- Water Jug Problem

## Approach

- Explore all possible states
- Use BFS for optimal solution
