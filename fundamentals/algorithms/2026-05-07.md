# Shortest Path Problem

Shortest path problems can be divided into multiple categories.

## 1. Single Source Shortest Path

Find:

- shortest distance from one start node
- to all other nodes

Example:
- Navigation
- Network routing

### Representative Algorithm

- Dijkstra Algorithm

---

## 2. All-Pairs Shortest Path

Find:

- shortest distance between every pair of nodes

### Representative Algorithm

- Floyd–Warshall Algorithm

---

# Dijkstra Algorithm

## Core Idea

Maintain:

- current known shortest distances

Then:

1. Select the nearest unvisited node
2. Finalize its distance
3. Update neighboring nodes

---

# Why Choose Minimum Distance Node?

If a node already has the smallest tentative distance:

→ No alternative path can produce a smaller value later.

Therefore:

- Its distance becomes final.

---

# Distance Update

For neighboring node `v`:

```python
dist[v] = min(
    dist[v],
    dist[u] + weight(u,v)
)
```

---

# Process

1. Initialize distances
2. Mark start node distance = 0
3. Repeat:
   - choose nearest unvisited node
   - relax edges
   - update distances

---

# Path Reconstruction

Distance alone does not preserve routes.

## Solution

Store:

- previous node (parent)

Example:

```python
parent[4] = 2
parent[2] = 3
```

Then:

```python
4 ← 2 ← 3 ← start
```

---

# Complexity

## Naive Implementation

Search minimum node every iteration

→ O(V²)

---

# Priority Queue Optimization

## Idea

Use:

- Min Heap / Priority Queue

Store:

```python
(distance, node)
```

Then:

- extract minimum efficiently

---

# 특징

- Same node may enter heap multiple times
- Smaller distance naturally processed first

---

# Floyd–Warshall Algorithm

## Goal

Find shortest paths:

- between all node pairs

---

# Core Idea

Gradually increase allowed intermediate nodes.

For every pair `(i, j)`:

Check whether going through `k` is shorter.

---

# Transition Formula

```python
D[i][j] = min(
    D[i][j],
    D[i][k] + D[k][j]
)
```

---

# Process

```python
for k:
    for i:
        for j:
```

## Meaning

- `k` = allowed intermediate node
- progressively expands usable routes

---

# Complexity

Three nested loops:

→ O(V³)

---

# Maximum Flow Problem

## Goal

Find:

- maximum possible flow
- from Source → Sink

---

# Conditions

Each edge has:

- capacity limit

Intermediate nodes satisfy:

```python
incoming flow = outgoing flow
```

---

# Residual Graph

## Very Important Concept

When flow is sent:

- create reverse edge

This enables:

- canceling previous choices
- rerouting flow

---

# Reverse Edge Meaning

A reverse edge represents:

> "Undo previously used flow"

This allows the algorithm to:

- improve earlier decisions
- discover better flow configurations

---

# Ford–Fulkerson Idea

## Process

1. Find any augmenting path
2. Send possible flow
3. Reduce forward capacity
4. Increase reverse capacity
5. Repeat until impossible

---

# Bipartite Graph

## Definition

Vertices divided into two groups:

- Left set
- Right set

Edges only connect:

- different groups

---

# Maximum Bipartite Matching

## Goal

Create:

- maximum number of non-overlapping pairs

---

# Key Insight

Maximum matching can be transformed into:

- Maximum Flow problem

Using:

- Source
- Sink
- Unit capacities

---

# Intuition

Reverse edges allow:

- canceling old matching
- rematching nodes
- improving total pair count
