# Topological Sort (DFS)

Topological sort determines an order of nodes given **dependency constraints**.

Example:
- Course prerequisites
- Task scheduling

## DFS Approach

### Core Idea

- Perform DFS
- **Record node when recursion finishes**
- Reverse the recorded order

### Steps

1. For all nodes:
2. If not visited → start DFS
3. Record node on return
4. Reverse the result

### 특징

- Multiple valid answers exist
- Depends on traversal order

# Minimum Spanning Tree (MST)

## Definition

A subgraph that:

- Includes all vertices
- Has no cycles
- Minimizes total edge weight

→ Called **Minimum Spanning Tree (MST)**

## Properties

- Multiple MSTs may exist
- Adding an edge → creates a cycle
- Removing an edge → disconnects graph

---

# Graph Cut & Crossing Edge

## Graph Cut

Partition vertices into two disjoint sets

## Crossing Edge

An edge that connects the two sets

## Key Property (Very Important)

> In any graph cut,  
> the **minimum-weight crossing edge is always part of MST**

# Prim's Algorithm

## Idea

- Grow MST step by step
- Always pick the **minimum edge connecting tree to outside**

## Process

1. Start from any node
2. Divide nodes into:
   - visited (in MST)
   - unvisited
3. Select minimum crossing edge
4. Add to MST
5. Repeat

## Implementation

### 1. Naive

- Check all edges each time  
→ O(NM)

### 2. Priority Queue (Heap)

- Store candidate edges
- Extract minimum efficiently

→ Preferred approach

## Key Insight

- Always expand using the **smallest outgoing edge**

# Kruskal's Algorithm

## Idea

- Build MST using edges
- Always pick **smallest edge first**

## Process

1. Sort edges by weight
2. Iterate edges:
3. If no cycle → add edge
4. If cycle → skip

## Data Structure

- Union-Find (Disjoint Set)

## Key Insight

- Avoid cycles while choosing smallest edges

---

# Intuition of MST

If a cycle exists:

→ Removing the **largest edge** keeps total weight minimal

⇒ Therefore:
- Kruskal naturally works by selecting smallest edges first

# MST Application (Water Supply Problem)

## Problem

Each node has:
- Cost to build a well
- Cost to connect to others

## Trick

👉 Add a **virtual node**

- Virtual node = "well source"
- Connect each node with well cost

## Result

Running MST automatically considers:
- Building wells
- Connecting nodes
- Mixed strategies
