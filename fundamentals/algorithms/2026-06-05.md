# Sweep Line Algorithm for Segment Intersection

## Definition

A computational geometry algorithm that detects line segment intersections efficiently by:

- Sweeping a vertical line from left to right
- Processing only important events
- Maintaining currently active segments
- Checking intersections only between neighboring segments

---

## Core Idea

### Naive Approach

Check every pair of segments.

For N segments:

:contentReference[oaicite:0]{index=0}

Problems:

- Many segment pairs can never intersect
- Huge number of unnecessary comparisons

---

### Sweep Line Idea

Instead of checking every pair:

1. Sort all segment endpoints by x-coordinate
2. Sweep from left to right
3. Maintain currently active segments
4. Check only adjacent segments

---

## Why Adjacent Segments Only?

Suppose active order is:

```text
3
0
1
```

and segment 1 is inserted.

We only check:

```text
0 ↔ 1
```

NOT

```text
3 ↔ 1
```

### Reason

For segment 1 to intersect segment 3:

```text
1 must first pass through 0
```

Therefore:

- Non-adjacent segments cannot suddenly intersect
- Only neighboring segments matter

> Very Important:
>
> A segment can only intersect another segment after becoming adjacent in the active set.

This observation removes most unnecessary comparisons.

---

## Event Processing

### Event Types

For each segment:

```text
Start Point
End Point
Intersection Point
```

All events are processed in x-coordinate order.

---

## Active Set

### Definition

Stores all segments currently crossed by the sweep line.

Segments are ordered by:

```text
Current Y-coordinate
```

Example:

```text
Top
↓
1
0
2
4
Bottom
```

---

## Process

### 1. Segment Start Event

Insert segment into active set.

Check:

```text
New Segment ↔ Previous Neighbor
New Segment ↔ Next Neighbor
```

Only these intersections are possible.

---

### 2. Intersection Event

When two neighboring segments intersect:

```text
A
B
```

becomes

```text
B
A
```

after crossing.

Therefore:

1. Swap their order
2. Check newly adjacent neighbors

---

### 3. Segment End Event

Remove segment from active set.

If removal creates new neighbors:

```text
A
Removed Segment
B
```

then check:

```text
A ↔ B
```

because they become adjacent for the first time.

---

## Key Insight

> Intersections can only occur between adjacent segments in the active ordering.

Thus:

- Ignore inactive segments
- Ignore non-neighboring active segments

Result:

Far fewer intersection tests.

# Finding Neighbor Segments in BST

After insertion we must know:

```text
Previous Segment
Next Segment
```

without performing a full inorder traversal.

---

## Previous Node (Predecessor)

### Case 1

Left child exists.

Move:

```text
Left Once
Then Right As Far As Possible
```

Example:

```text
Current
 /
Subtree
```

Largest node inside left subtree
= predecessor

---

### Case 2

No left child.

Move upward.

Find first ancestor reached from:

```text
Right Child
```

That ancestor becomes predecessor.

---

## Next Node (Successor)

Symmetric logic.

### Case 1

Right child exists.

Move:

```text
Right Once
Then Left As Far As Possible
```

Smallest node in right subtree
= successor

---

### Case 2

No right child.

Move upward.

Find first ancestor reached from:

```text
Left Child
```

That ancestor becomes successor.

---

## Key Insight

Predecessor:

```text
Largest node smaller than current node
```

Successor:

```text
Smallest node larger than current node
```

Both can be found in:

:contentReference[oaicite:2]{index=2}

using BST parent links.

# Why BST Is Used

## Intuitive View

Need to repeatedly perform:

- Insert segment
- Delete segment
- Find previous segment
- Find next segment

All efficiently.

---

## BST Operations

| Operation | Complexity |
|------------|------------|
| Insert | O(log N) |
| Delete | O(log N) |
| Find Predecessor | O(log N) |
| Find Successor | O(log N) |

Perfect match for Sweep Line.

---

## Naive Intersection

## Sweep Line

Bentley-Ottmann style:

```text
Event Processing:
O((N + K) log N)
```

where:

- N = number of segments
- K = number of intersections

Much faster when K is small.

# Notes

### Very Important

- Active Set is ordered by current Y-position.
- Only adjacent segments are tested.
- When intersection occurs:
  - swap order
  - check new neighbors
- When segment disappears:
  - newly adjacent segments must be checked.
- BST enables efficient insertion and neighbor search.

# Exam Tips

### Frequently Asked Questions

#### Why are only neighboring segments checked?

Because:

```text
To intersect a non-neighbor segment,
a segment must first become adjacent to it.
```

---

#### Why use BST?

Because active segment ordering changes dynamically.

Need:

```text
Insert
Delete
Predecessor
Successor
```

all in O(log N).

---

#### Difference from Brute Force?

Brute Force:

```text
Check every pair
```

Sweep Line:

```text
Check only potentially intersecting pairs
```
