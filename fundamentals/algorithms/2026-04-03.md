# Shell Sort

Shell Sort는 **삽입 정렬을 개선한 알고리즘**이다.

## 핵심 아이디어

- 거의 정렬된 데이터에서 빠름
- 간격(h)을 두고 부분 정렬 수행
- h를 점점 줄이며 정렬

## 과정

1. 큰 간격(h)으로 그룹 나누기
2. 각 그룹을 삽입 정렬
3. h를 줄이면서 반복
4. 마지막 h = 1 → 일반 삽입 정렬

## 장점

- 큰 값은 뒤로, 작은 값은 앞으로 빠르게 이동
- 삽입 정렬의 단점 보완

# Linear Sorting

## Radix Sort

- 자릿수 기준 정렬
- 뒤 자리 → 앞 자리 순으로 진행
- **안정 정렬 필수**

## Counting Sort

- 값의 범위를 기반으로 카운팅 배열 사용
- 누적합을 통해 정렬

## 특징

- 데이터 범위에 따라 성능 좌우됨
- 비교 기반 정렬이 아님

## 정의

> 모든 경우의 수를 전부 탐색하는 방법

# Graph Search

그래프는 매우 중요한 자료구조

---

## DFS (Depth-First Search)

### 특징

- 갈 수 있는 만큼 깊이 탐색
- 스택 / 재귀 사용

### 흐름

1. 현재 노드 방문
2. 인접 노드 탐색
3. 끝까지 가면 되돌아감 (Backtracking)

---

## BFS (Breadth-First Search)

### 특징

- 같은 거리 노드 먼저 탐색
- 큐 사용

### 흐름

1. 시작 노드
2. 인접 노드 모두 방문
3. 다음 레벨 탐색

---

# DFS vs BFS

| 구분 | DFS | BFS |
|------|-----|-----|
| 방식 | 깊이 우선 | 너비 우선 |
| 자료구조 | Stack / Recursion | Queue |
| 특징 | 깊게 탐색 | 가까운 것부터 |

---

# 핵심 정리

- 완전 탐색 = 모든 경우 확인
- 탐색 공간은 순열 / 조합 / 부분집합으로 확장됨
- 조합 최적화 문제는 계산량이 매우 큼
- 그래프 탐색 핵심 = DFS, BFS
