# Shell Sort

Shell Sort is an **improved version of Insertion Sort**.

## Key Idea

- Performs well on nearly sorted data
- Uses a gap (h) to partially sort elements
- Gradually reduces the gap

## Process

1. Divide the array into groups using a large gap (h)
2. Apply insertion sort within each group
3. Reduce the gap and repeat
4. Final step: h = 1 → standard insertion sort

## Advantages

- Moves large values backward and small values forward efficiently
- Overcomes the limitations of insertion sort

# Linear Sorting

## Radix Sort

- Sorts numbers based on digit positions
- Processes from least significant digit to most significant digit
- Requires a **stable sorting algorithm**

## Counting Sort

- Uses a counting array based on value ranges
- Uses prefix sums to determine positions

## Characteristics

- Performance depends on the range of input values
- Not a comparison-based sorting algorithm

## Key Points

# Greedy: Meeting Room Problem

## Problem

- Select the maximum number of non-overlapping meetings

## Key Strategy

👉 **Always select the meeting that ends earliest**

## Solution Approach

1. Sort meetings by end time
2. Select meetings sequentially if they do not overlap

---

# Two Pointer: Solution Mixing Problem

## Problem

- Choose two solutions such that their sum is closest to 0

## Key Idea

- Start from both ends of the array
- Adjust pointers based on the sum

## Strategy

- If sum > 0 → move right pointer left
- If sum < 0 → move left pointer right

# Brute Force

## Definition

> A method that explores all possible cases

## Characteristics

- Very simple to implement
- Leverages computational power
- Inefficient when the search space is large

## Considerations

- Size of the search space
- Cost per operation

---

# String Matching (Brute Force Example)

## Problem

- Find pattern P in string T

## Approach

1. Move the starting position one by one
2. Compare the pattern with substring
3. Return the index if matched

# Types of Search Space

## 1. Permutation

- Order matters
- Total cases: n!

## 2. Combination

- Order does not matter
- Eliminates duplicates

## 3. Subset

- Include or exclude each element
- Total cases: 2ⁿ

# Combinatorial Optimization

> Optimization problems based on combinations

---

## 1. TSP (Traveling Salesman Problem)

- Visit all nodes and return to the starting point
- Find the shortest path

### Characteristics

- Number of cases: n!
- Practically impossible with brute force

---

## 2. Knapsack Problem

- Maximize value within a weight limit

### Characteristics

- Number of cases: 2ⁿ
- Can be solved using Dynamic Programming

---

## 3. Assignment Problem

- Assign tasks to minimize total cost

### Characteristics

- Permutation-based problem

# Graph Search

Graphs are a very important data structure.

---

## DFS (Depth-First Search)

### Characteristics

- Explores as deep as possible first
- Uses stack or recursion

### Flow

1. Visit current node
2. Explore adjacent nodes
3. Backtrack when no more nodes are available

---

## BFS (Breadth-First Search)

### Characteristics

- Explores nodes level by level
- Uses a queue

### Flow

1. Start from the initial node
2. Visit all adjacent nodes
3. Move to the next level

---

# DFS vs BFS

| Type | DFS | BFS |
|------|-----|-----|
| Strategy | Depth-first | Breadth-first |
| Data Structure | Stack / Recursion | Queue |
| 특징 | Deep exploration | Closest nodes first |

# Key Takeaways

- Brute force explores all possible cases
- Search space expands into permutations, combinations, and subsets
- Combinatorial optimization problems have very high complexity
- Core graph traversal techniques: DFS and BFS
