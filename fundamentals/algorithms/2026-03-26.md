## Quick Sort

Quick Sort is a divide-and-conquer sorting algorithm.

It selects a pivot and partitions the array based on that pivot.

### Idea
- Choose a pivot
- Place smaller elements on the left
- Place larger elements on the right
- Recursively apply the same process

### Process
1. Select a pivot (e.g., last element)
2. Partition array into two parts
3. Place pivot in correct position
4. Recursively sort left and right

### Features
- In-place sorting (no extra array needed)
- Very efficient in practice

### Time Complexity
- Average: O(n log n)
- Worst: O(n²)

### Worst Case Occurs When
- Pivot is always smallest or largest
- Input is already sorted

### Improvement
- Random pivot selection
- Median-of-three method

## Fibonacci Problem

### Naive Recursion

F(n) = F(n-1) + F(n-2)

### Problem
- Same values are repeatedly computed
- Very inefficient

### Time Complexity
- O(2ⁿ)

### Solutions

#### 1) Dynamic Programming
- Store computed values
- Avoid repeated calculations

#### 2) Iterative Approach
- Compute sequentially
- Most practical for small values

#### 3) Matrix Method
- Uses matrix exponentiation
- F(n) can be computed in O(log n)

## Sorting Algorithm Properties

### 1. In-place Sorting
Uses very little extra memory.

**Examples:**
- Quick Sort
- Insertion Sort

### 2. Stable Sorting
Maintains the relative order of equal elements.

**Important when:**
- Sorting multi-key data

## Selection Sort

### Idea
Select the maximum (or minimum) and place it at the correct position.

### Process
1. Find the largest element
2. Swap with the last element
3. Repeat for remaining elements

### Time Complexity
- O(n²) (always the same regardless of input)

### Properties
- In-place: O
- Stable: ❌

---

## Bubble Sort

### Idea
Compare adjacent elements and swap if needed.

### Process
- Repeatedly move the largest element to the end
- Like bubbles rising

### Time Complexity
- O(n²)

### Properties
- In-place: O
- Stable: O

---

## Insertion Sort

### Idea
Build a sorted portion step by step.

### Process
1. Assume first element is sorted
2. Insert next element into correct position
3. Repeat

### Time Complexity
- Best: O(n) (already sorted)
- Worst: O(n²)

### Properties
- In-place: O
- Stable: O

## Key Takeaways

- Quick Sort is the most practical general-purpose sort
- Selection Sort is simple but inefficient
- Bubble Sort is stable but slow
- Insertion Sort is efficient for nearly sorted data
