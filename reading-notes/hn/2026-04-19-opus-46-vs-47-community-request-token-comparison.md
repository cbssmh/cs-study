# Opus 4.6 vs Opus 4.7 Request Token Comparison

출처: billchambers.me  
성격: 커뮤니티 익명 제출 기반 비교 데이터  
주의: Anthropic 공식 자료가 아니며, "Not affiliated with or endorsed by Anthropic"라고 명시되어 있음.

## 요약

커뮤니티 제출 476건 기준으로, **Opus 4.7은 Opus 4.6 대비 요청당 평균 토큰 수가 37.4% 증가**했다.  
표에 따르면 **요청 비용 증가율도 동일하게 평균 37.4%**로 나타난다.

- **AVG REQUEST TOKEN CHANGE**: +37.4%
- **AVG REQUEST COST CHANGE**: +37.4%
- **표기된 AVG REQUEST SIZE**: 349 / 466
  - 해석상 4.6 평균 349 tokens, 4.7 평균 466 tokens로 보임

즉, 같은 입력에 대해 커뮤니티 샘플 기준으로 보면 **Opus 4.7이 더 많은 request tokens를 소비하는 경향**이 있다.

## 관찰 포인트

### 1. 비용 증가는 사실상 토큰 증가와 동일
표에 나온 사례들은 request token 증가율과 cost 증가율이 항상 동일하다.  
이는 이 비교가 **request token 단가를 고정한 상태에서 토큰 수 차이만 반영**하고 있음을 시사한다.

예시:
- 12 → 17 tokens: **+41.7%**
- $0.000180 → $0.000255: **+41.7%**

### 2. 작은 요청에서 증가폭이 크게 보이는 사례가 많음
최근 50개 예시를 보면 작은 요청에서 증가율이 특히 크게 보인다.

예:
- 9 → 16: **+77.8%**
- 34 → 57: **+67.6%**
- 65 → 112: **+72.3%**
- 8 → 13: **+62.5%**

작은 절대값 변화도 퍼센트로는 크게 나타날 수 있으므로,  
**작은 요청의 높은 증가율만 보고 일반화하면 왜곡될 수 있음**.

### 3. 큰 요청에서도 증가하는 사례가 존재
단순히 소형 요청만의 현상은 아니다.

예:
- 3,725 → 5,411: **+45.3%**
- 222 → 319: **+43.7%**
- 450 → 643: **+42.9%**
- 464 → 621: **+33.8%**

즉, 일부 대형 요청에서도 4.7이 의미 있게 더 큰 request size를 보였다.

### 4. 모든 요청이 크게 늘어나는 것은 아님
증가폭이 매우 작은 사례도 있다.

예:
- 605 → 610: **+0.8%**
- 1,154 → 1,159: **+0.4%**
- 173 → 177: **+2.3%**
- 193 → 197: **+2.1%**

따라서 **4.7이 항상 크게 증가하는 것은 아니고**,  
프롬프트 유형에 따라 차이가 큰 것으로 보인다.

## 해석

이 데이터만 놓고 보면, 커뮤니티가 제출한 실제 입력들에서는  
**Opus 4.7이 Opus 4.6보다 request token을 더 많이 쓰는 경향이 강하다**.

다만 아래 한계가 있다.

- 커뮤니티 익명 제출 기반이라 표본이 통제되지 않음
- 최근 50개 표는 전체 476건의 분포를 대표하지 않을 수 있음
- 프롬프트 종류, 시스템 프롬프트 유무, 툴 사용 맥락 등이 공개되어 있지 않음
- 공식 벤치마크가 아니라 관찰용 데이터에 가까움

## 실무적으로 볼 점

이 자료는 다음 판단에 참고할 수 있다.

- **비용 민감한 워크로드**에서는 4.7 전환 시 request 비용 상승 가능성 검토 필요
- 특히 **짧은 요청에서도 상대 증가율이 크게 튈 수 있음**
- 반면 일부 요청은 차이가 거의 없으므로, 실제 도입 전에는 **자체 프롬프트 셋으로 샘플링 테스트**가 더 중요함

## 한 줄 결론

커뮤니티 비교 476건 기준으로 **Opus 4.7은 Opus 4.6 대비 request tokens와 request cost가 평균 약 37.4% 증가**한 것으로 나타난다. 다만 비공식·비통제 표본이므로, 의사결정에는 자체 검증이 필요하다.
